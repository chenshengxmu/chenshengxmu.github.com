本文来自 TECH2IPO / 创见，作者@DavidZh?

2016 年 Google I/O 大会开放注册之后，Google 出人意料地放出了下一代移动操作系统 Android N 的开发者版本。

包括 The Verge 在内的多家国外科技媒体抢先上手了 Android N 开发者预览版。与目前市占率只有 2.3% 的 Android M（M For Marshmallow）相比，Android N 最直观的改进就是增加了系统级别的分屏多任务。

根据 Google 公布的新版开发者导引，Android N 支持两种分屏模式。一是双窗口并排显示，第二种是一个 App 界面浮在另一个 App 之上。在安装了 Android N 的智能电视上，还支持画中画视频播放（Picture in Picture）——iPad Pro 静静地看着你 Google。

关注手机操作系统尤其是国产定制 ROM 的朋友，对「分屏多任务」这个词想必不会感到陌生。进入 2015 年之后，各家厂商的屏幕尺寸纷纷突破 5 吋，更有意思的是，为了照顾手小用户的感受，定制 ROM 们也不得不在系统中预置「单手模式」。尽管这并不是最佳的解决方法，但聊胜于无，只不过有一部分屏幕空间被浪费掉了。

分屏多任务几乎可以完美解决这个问题。vivo 在 2015 年底发布 X6 和 X6 Plus 时，就推出了类似功能。可惜由于 Android 5.0 系统底层并不支持分屏，vivo 的 Funtouch OS 只能让系统短信、微信、QQ、系统视频、爱奇艺、优酷等有数几个 App 支持这一特性。

Android N 携分屏多任务而来，证实了用户在大屏手机时代的真实需求，也预示着 Android 具备了一个优秀生产力操作系统的潜质。

除了分屏多任务，Android N 在通知中心也进行不小改动

第一个重大变化是，借鉴了 iOS 系统的通知栏快捷回复特性。通知下拉到第一个层级时，会优先显示常用的快捷开关比如 Wi-Fi、手电筒、数据网络和电池等。这个特性大概是由三星的 TouchWiz 而来。

另一个改动是，Android N 可以根据来源将接收到的通知归拢在一起，而不是像现在这样所有通知散落成一堆。通知中心的电池图标也具备了新的功能，不再只能显示剩余电量，点进去还能直接在通知中心产看电量消耗情况，非常方便。

除了这些交互界面上可见的变化，Android N 还在后台内置了数据节省模块（Data Saver），用户可以自定义在使用 3G/4G 网络时能够联网的 App 名单。

对于开发者而言，Android N 在 SDK 上也带来一些提升。最让程序员 GG 高兴的莫过于支持 Java 8，撸代码再也不像从前那样费劲了。另外，早在此前 Google 就宣称新的操作系统将开始采用开源的 OpenJDK 和 Java API 接口，会在执行效率上带来提升。

虽然今年的 I/O 大会尚未召开，Google 仍然希望尽早获得新系统尝鲜者的反馈，于是推出了 Android Beta 项目。与微软的 Windows Insider 相似，尝鲜者需要拥有 Android N 已支持的设备，并到 Android Beta 官网注册，才能将新系统安装到设备中同时接受后续的系统更新。
