title: 移动Web体验糟糕，谁之过？

自从 Apple 推出了 Apple News，而 Facebook 推出了 Instant Articles 之后，欧美的科技媒体就开始了口诛笔伐。除了批评公司们强加给用户调用的各种广告代码，以及移动浏览器 Web 的糟糕体验之外，也对其背后的形成原因进行了分析。对于用户来说，一台 2007 年的 MacBook Pro 和一部 2014 年的 iPhone 6 在访问同一网址时，前者的速度竟然更快——虽然理论上后者的硬件计算能力更强。让我们来看看这究竟是怎么一回事：

Internet Explorer 曾经是 Windows 系统内置的一款优秀的浏览器，在微软垄断了操作系统市场，并借由 Windows 的普及强推 IE 之后，这款曾经战胜了 Navigator Netscape 的产品一直止步不前，且不断被人诟病。在 Mozilla Firefox 和 Google Chrome 面世之后，Internet Explorer 基本已经失去了竞争力，甚至成为了业界和用户的笑柄。造成这种局面的原因之一，就是 Internet Explorer 曾经极高的市场占有率，让微软失去了改进这款产品的动力。

桌面互联网时代的这一幕，正在移动互联网时代重演。

在苹果 iPhone 遍布全球的今天，这家公司却禁止在 iOS 平台上出现第二种浏览器引擎，使得 iOS 版的 Safari 成为用户唯一的选择。科技媒体 TheVerge 的编辑 Nilay Patel 做了一个测试，结果发现：一台 2007 年的 MacBook Pro 和一部 2014 年的 iPhone 6 在访问同一网址时，前者的速度竟然更快。以至于他最终给出了一个结论：iPhone 上的 Safari 就是当年的 Internet Explorer。

Nilay Patel 在文章中指出，正是因为移动 Web 浏览器阅读体验的糟糕，导致 App 在移动端的蓬勃发展。他对比了桌面端 App 的发展情况，指出微软希望从 Windows 8.1 开始打造一个桌面端的 App 生态，但是用户并不买账，因为桌面端的 Web 浏览器足够好，导致对于 App 的需求并不高。需要设计一款 App 来满足的需求，开发一个 Web 应用就可以了。

反观移动端，倒是苹果曾经力捧的 Web App 始终没有盛行，甚至 Facebook 都承认在这一技术选择上有失误。相对封闭的 App 的流行，也在一定程度上导致了移动互联网的封闭局面。公司纷纷推出自有的专用 App，而开放的 Web 标准则无人问津。

Nilay Patel 在文章中还谴责了移动互联网时代，IT 公司们无所不用其极的广告代码。这些广告代码是用户在调用一个网页是被迫加载的，使用的是用户购买的网络带宽和计算设备，而且还要用户贡献时间和注意力。在 Web 发展的早期，虽然也存在广告代码，但是和今天庞大的广告系统相比，简直是小巫见大巫。如今的数字广告是一个规模 50 亿美元的市场，广告代码是这个行业赚钱的工具之一，各种新兴的数字广告展示方式和程序化购买系统，需要更加复杂的广告代码在用户的电脑上运行，对浏览器的性能要求更高。

而苹果的 News 和 Facebook 的 Instant Articles，除了如虎嗅之前的文章所说，是在“封闭花园”中复制互联网之外，也是因为目前互联网广告代码已经对于内容阅读体验产生了严重的负面效果所致。

在一些行业人士眼中，数字广告已经触发了众怒。“我们距离在线广告消失的那一刻只有不到一分钟。”Mozilla 基金会负责广告业务的内容服务副总裁 Darren Herman 在一篇文章中这样写道，“广告和用户，两者之间的关系越来越紧张，这种感觉比以往任何时候都要强烈。”

究其原因，互联网用户如今屏蔽广告内容的工具多种多样，浏览器屏蔽广告插件的用户安装率非常高。Herman 在文章中指出，有行业分析估计的数据显示，广告屏蔽插件以每年 43% 的增幅在迅速普及。

不仅是互联网广告，就连电视广告都是如此。

Herman 在文章中透露，数字内容提供商 TiVo 于 2014 年在美国进行的一项行业研究显示，电视剧《广告狂人》的观众当中有 73％ 会跳过广告，《行尸走肉》的观众有 66％ 会跳过广告。尽管目前在美国市场，数字广告的预算要比电视广告低 49％，但是数字广告的购买方式却有很多 (CPM、CPE、CPC、CPA 和 CPL 等等)，不过在广告屏蔽工具面前，就好像用户跳过电视广告一样，无能为力。

在 2015 虎嗅 F&M 创新节上海站上，分众传媒江南春为我们描绘了广告的终极形态：

以后可以想象一个事情，我们在电梯里面，给每块屏幕投放的广告是不一样的。我在机器里面有Wifi，你会感应到这个人走进来，你只要开着Wifi我就能感应到你的地址，我就可以感应到你的云端。这个地址在过去一个月马云淘宝天猫上买过什么东西，我就发了一个指令，这个里面存了300个广告，当你靠近我的时候我给你推哪个广告，其实这在我们系统当中是很容易实现的事情。

为了这种终极形态，我们看到互联网公司在竭力发展各自的广告系统，甚至浏览器也要为搜集用户数据贡献一份力 (即便有安全隐患也在所不惜)。

可是，这种终极形态真的会来到吗？
