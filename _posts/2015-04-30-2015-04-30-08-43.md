北京时间今天凌晨，微软Build2015开发者大会在美国旧金山开幕，首日看点不少：微软计划在 Windows 10 发布 2-3 年内实现驱动 10 亿部设备，取代IE的新浏览器被正式命名为Microsoft Edge，Android 和 iOS 应用可轻松移植为 Windows 10 通用应用，Office将集成Uber叫车等更多功能，Windows 10 手机版 Continuum 功能能让手机一秒变 PC，全新的Visual Studio Code 编辑器免费支持 Windows、Mac 和 Linux三大平台。

在开发大会上，微软Windows业务的负责人迈尔森（Terry Myerson）做出了这番表态，他表示这是“两年或三年之内”希望达到的渗透率目标。迈尔森也表示，没有其他生态系统的平台（操作系统）能够覆盖十亿台设备。微软的Windows 10同时兼容个人电脑、智能手机、平板电脑、游戏机等多种硬件设备，因此10亿台设备的目标包括了多种类型的产品。

做个简单对比，Windows 8 花了 15 个月才实现 2 亿销量。

微软还在开发大会上宣布了 Project Spartan 的正式名，正式名为Microsoft Edge，将替代 IE 浏览器，而所用的引擎也叫 Edge。

主要功能和之前宣布的 Project Spartan 功能相似，除了主打的 3 大特性：笔记和共享、Cortana、无干扰阅读模式之外，Microsoft Edge 还支持 Chrome 和 Firefox 扩展插件的快速移植，“少量代码修改”就可以兼容。

关于微软 Windows 10将兼容 Android 应用的传言已经出现近一年，微软在 Build 2015 上给出了正式答案，Windows 10不会变成android模拟器，但Android 和 iOS 应用可移植为 Windows 10 通用应用。

Android 方面，微软实现让 Android 应用开发者复用 Java 和 C++ 代码。微软为 Windows 10 手机版系统增加了“一个子系统”来允许运行 Android 应用。有人会将这理解为“模拟器”，但微软更多地将这个机制称为“子系统”，这里面有模拟器的功能。

iOS 方面，微软开发了 iOS Objective-C 编译器，可以让 iOS 开发者将他们的应用重新编译为 Windows 10 手机版兼容的应用。这个编译器今天已经推出。

一些重要开发商已经在测试微软新的移植工具，包括 Candy Crush Saga 开发商 King。而且，从 Android 和 iOS 移植过来的 Windows 10 应用都将是通用应用，并能够加上微软 Windows 的特性，比如 Cortana、Xbox LIVE、动态磁贴等。

这一方面能丰富Windows 10 手机版应用的生态，但另一方面，肯定会打击开发者开发Windows 10 手机版原生应用的热情，既然能移植，那学学更有市场的iOS或android开发不是更省心。

微软在大会上展示了最新一代的Office产品，并且有意将Office打造成为一个新平台。 　　 未来Office 2016将会公开更多的应用接口，以方便更多的第三方服务进行接入。根据现场的演示，Office 2016将支持LinkedIn、Saleforce以及Uber插件。这将允许用户无需在不同应用和数据服务之间来回切换，生产力得到大幅提高。在今天的演示环节中，用户可以直接在Outlook的日历上查看到Uber提醒，或者直接呼叫Uber服务等。 　　 微软对于Office 2016保持了前所未有的开放，未来Office 2016的所有功能将会同时支持Windows和iOS平台。

微软还演示了一项有大幅改进的功能，Windows 10 手机版的 Continuum 功能，可以直接将手机连接到大屏幕上变为一台拥有 PC 体验的设备，直接让生产力翻倍。在现场演示中，Windows 10 手机版直接会把系统变为桌面体验，支持键鼠操作，开始屏幕变为开始菜单，手机版 Office 在进入 Continuum 模式后，直接变为平板版，拥有完整的界面。不过该功能需要新设备的支持。

微软还宣布了 Visual Studio Code 编辑器，支持 Windows、Mac 和 Linux，目前已经提供下载。

微软将 Visual Studio Code 描述为“代码优化编辑器”，支持 Intellisense（代码智能补全）、Debug、GIT，运行非常快速。

Visual Stuido 2015 被很多人认为是世界上最优秀的 IDE（集成开发环境），跨平台又免费的Visual Studio?Code 应该能够替代一些轻量级的编辑器。
